Mình đã tạo blog này như thế nào?

  Nhận được lời động viên và giúp đỡ từ chú runi mình đã tập tành viết blog các kiểu nhằm chia sẻ đến mọi người những gì mình vừa mới tìm hiểu được (mặc dù là văn chương mình rất tệ 😪). Và đây là bài blog đầu tiên của mình 😆

  Như mục đính chính của mình là chia sẻ những gì mình học được vì thế mình sẽ chia sẻ về cách mà mình tạo ra cái blog này :))))
    
  Blog này mình dùng framework hexo cung cấp bởi nodejs vì thế đối với các bạn lần đầu sử dụng nodejs thì cần phải cài nodejs.
  * Đối với các bạn dùng windows chúng ta có thể tải và cài đặt nodejs ở đây
  * Đối với các bạn dùng linux thì chúng ta dùng nvm để cài nodejs bằng lệnh:


Sau khi cài đặt xong chúng ta bắt đầu tạo blog thôi 😆

  1. Cài đặt hexo bằng câu lệnh:

  2. Khởi tạo project mới:

    Sau khi thực thi lệnh xong, hexo sẽ khởi tạo cho chúng ta 1 thư mục tương ứng với tên khi chúng ta khởi tạo.

  3. Start server
    Mặc định ban đầu server sẽ chạy ở http://localhost:4000

    Và đây là giao diện blog của chúng ta.

Để có cho blog nhìn đẹp hơn thì mình nên cài thêm các theme (blog mình đang dùng là theme icarus)

   Mọi người có thể xem thêm các theme khác ở đây

  1. Clone icarus từ github

  2. Chuyển blog sang theme icarus

  3. Cài cheerio dependencies

  4. Start server
    Dùng phím Ctrl+C để tắt server đạng chạy. Sau đó start lại


    Để chỉnh tên, tiêu đề, địa chỉ của chúng ta trên theme thì chỉnh phần widgets trong /themes/icarus/_config.yml

  5. Bắt đầu viết blog
    Các bài blog của chúng ta sẽ được viết trong source/_posts/, các file blog sẽ có định dạng là **.md. Đây là file markdown, khá là dễ Viết. Mọi người có thể tham khảo cách viết markdown file tại đây

  • Lưu ý: ở đầu file .md chúng ta nên thêm title,tags, categories, thumbnail
  1. Deploy blog lên github

  • Tạo 1 Repository với Repository name là <username>.github.io. VD: github username của mình là “vany-chocolate” thì mình sẽ tạo repository name là “vany-chocolate.github.io”

  • Cài deploy pakages cho hexo

  • Config deploy
    repo thay repository bằng link repository mới tạo

  • Deploy blog

Sau khi deploy xong mở link theo tên Repository chúng ta tạo và thưởng thức blog của chính mình nào 😍😍
